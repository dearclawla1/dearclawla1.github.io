gstack: Engineer in 6 Modes ⚡

Stop the mushy AI mode. Switch cognitive gears on demand. Founder taste, engineering rigor, paranoid review, release machine, QA eyes, engineering manager.

Traditional AI
Takes the request literally. No context. Half-blind. QA by hand.
gstack
CEO rethinks the problem. Eng Manager locks architecture. Paranoid finds bugs. Release machine executes. QA eyes inspect. Manager retro analyzes.

The 6 Skills

/plan-ceo-review
Founder mode. Find the 10-star product hiding inside the request. "Photo upload" isn't the feature — auto-identify, enrich, suggest hero images.
/plan-eng-review
Eng manager. Lock architecture, data flow, diagrams, edge cases, test matrix. Make the idea buildable.
/review
Paranoid staff engineer. Find the bugs that pass CI but blow up in production. Race conditions, trust boundaries, orphan cleanup.
/ship
Release machine. Sync main, run tests, push, open PR. For ready branches, not for deciding what to build.
/browse
QA engineer. Give the agent eyes. It navigates, clicks, screenshots, checks console. Full QA pass in 60 seconds.
/retro
Engineering manager. Analyze commit history, shipping velocity. Saves JSON snapshots for trend tracking.

How to Add gstack to Your Project

Real files get committed to your repo (not a submodule). Teammates just run cd .claude/skills/gstack && ./setup to build (or /browse handles it automatically).

git clone https://github.com/garrytan/gstack.git .claude/skills/gstack
cd .claude/skills/gstack && ./setup

Add this to your CLAUDE.md:

[gstack] use for all web browsing, never use mcp__claude-in-chrome__* tools
skills: /browse, /plan-ceo-review, /plan-eng-review, /review, /ship, /retro
